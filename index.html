PROFILE

テストの点数では測れない OS をつくるラボ

RTX 5080 ローカル推論と gpt-oss 系 LLM MCP ベースのエージェント SwiftUI と Blender VFX でインターフェースを組み合わせて 自分の思考と一緒に成長する ORA OS を設計する場所が ORA Hyperwave

学校の時間割ではなく 自分の脳内ロードマップをベースに プロトタイプが出来た瞬間から世界に公開するスタイルで開発を続けている

ORA LLM Stack
CORE PIPELINE

RTX 5080 ローカル推論と MCP オーケストレーション

gpt-oss 20B と 120B クラスのモデルをローカルで回しながら 必要に応じてクラウドノードをブリッジ MCP エージェントが VFX ツールと SwiftUI プロジェクトとブラウザを横断して制御する

Hyperwave では チャットからシーン生成 Git コミットまでをひとつのタイムラインに並べて ログを Hyperstream として公開することで ただの作品集ではなく 進化し続ける OS の変遷として見せていく

HARDWARE

Hybrid Compute Bank

RTX 5080 ノードを軸に 推論 レンダリング HTTP 経由のツール呼び出しを同じマシンで完結させる構成 レイテンシを削りながら ローカルファイルと GPU を全力で使う前提で設計

AGENTS

ORA MCP Mesh

コード生成 VFX パラメータ調整 SwiftUI レイアウト提案など 役割を分けた MCP エージェント群が チャット一往復で動けるようにパイプラインを整理中

TELEMETRY

Hyperstream Metrics

X Discord YouTube GitHub の反応をまとめて どの実験が一番刺さっているかを軽くチェック 高い反応のユニットから順に Hyperwave の上段に固定していく運用

Liquid SwiftUI Lab
LIQUID DESIGN

液体ガラスみたいに揺れる UI コンポーネント

Metal シェーダと SwiftUI Custom Layout を組み合わせて 触るたびにほんの少しだけ揺れる Liquid Glass UI を試作 iPhone iPad Mac そして Vision Pro まで一貫した質感を狙う

MULTI PLATFORM

ORA Mobile と ORA Desktop クライアント

ORA Mobile Beta で日常の対話とタスクをまとめ Windows デスクトップ側では重い推論と VFX を担当 それを Hyperwave Portal からひとつのブランドとしてつなぐ

MATERIAL

Dynamic Color Fields

環境光とコンテンツに応じて グラデーションとブラー量を少しだけ変化させるマテリアルシステム Apple のガラス感に近づけつつ ORA らしい色を重ねていく

PROTOTYPING

Zero Delay Deploy

GitHub Actions とビルドスクリプトで UI の変更をできるだけ短いループでデバイスに反映 思いつきをその日のうちに試作して捨てる前提のプロトタイピング

ASSIST

LLM Assisted SwiftUI

LLM にコンポーネントのたたき台を出してもらい 最後の細かいトランジションや階層構造は自分で詰めるスタイル 人間の感覚と AI の速度を両方使う前提のデザインフロー

VFX Nodes Library
PROCEDURAL BANK

未来都市の空気感をつくるノード群

Blender のジオメトリノードとマテリアルノードで プラズマ 霧 衝撃波などのプリセットを量産 MCP エージェントがパラメータを振りながら WebGL や映像に書き出す流れを構築中

BURST

Procedural Shockwaves

オーディオのリズムとカメラモーションから波形を生成して UI とシンクした衝撃波を描画 ORA Hyperwave のメインビジュアルとして使えるレベルまで調整していく

FOG

Volumetric Streets

微細な粒子とボリュームライトで 夜の未来都市の空気感を作るプリセット ただの背景ではなく UI の奥行きとして使う前提で設計

PORTAL

Hyperwave Gateway

ページ遷移やシーン切り替えを VFX ポータルとして見せるトランジション Apple っぽい上品さを残しつつ 少しだけサイバーに振った表現を研究中

Signal Log and Ops

運用ルール

週に一度 PV とクリック率をざっくり眺めて よく押されているリンクとプロトタイプを上段に固定 反応が薄い部分はテキストやサムネを差し替えたり 下段に移動させたりして調整する

ミッション

ORA Hyperwave の目的は 未来の OS を数年早く触っている感覚を届けること 完成したものだけではなく 思考と失敗も含めて公開することで 個人開発と学習のログをそのままポートフォリオにしていく

アクセスチャネル
